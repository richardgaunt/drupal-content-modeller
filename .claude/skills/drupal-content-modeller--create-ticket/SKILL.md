# Complete Ticket Skill

You are helping the user complete a partially-filled QA ticket markdown file. The ticket follows the format generated by `dcm report templates` and describes a Drupal entity type bundle with fields, widgets, and permissions.

## Step 1: Read the ticket file

The user will provide a path to a markdown file. Read it.

## Step 2: Extract the entity type

Look for the HTML comment `<!-- entity_type: <type> -->` in the file. Valid types: `node`, `media`, `paragraph`, `taxonomy_term`, `block_content`.

If not found, infer from context:
- Title says "content type" → `node`
- Title says "media type" → `media`
- Title says "paragraph type" → `paragraph`
- Title says "vocabulary" → `taxonomy_term`
- Title says "block type" → `block_content`

## Step 3: Get project info (if available)

If the user mentions a project, run:
```
dcm field types --json
dcm field prefixes --json
```

This gives you the valid field types and prefix for the entity type.

## Step 4: Complete the fields table

For each row in the fields table, apply these defaults for any empty cells:

### Machine Name (if empty)
Generate from the Field Name:
1. Lowercase, replace spaces with underscores, strip non-alphanumeric
2. Prepend the entity type field prefix:
   - node: `field_n_`
   - media: `field_m_`
   - paragraph: `field_p_`
   - taxonomy_term: `field_t_`
   - block_content: `field_b_`

If the project uses a custom prefix (like `field_c_n_` for CivicTheme), check existing fields in the project to detect the pattern and use it.

### Field Type (if empty)
Infer from the Field Name using common patterns:
- "image", "photo", "picture", "thumbnail" → `image`
- "file", "document", "attachment" → `file`
- "body", "content", "summary", "text" (long text) → `text_long`
- "title", "name", "label", "heading" (short text) → `string`
- "description", "intro", "teaser" → `string_long`
- "link", "url", "website" → `link`
- "email", "e-mail" → `email`
- "date", "start date", "end date" → `datetime`
- "date range" → `daterange`
- "number", "count", "quantity", "weight", "order" → `integer`
- "price", "amount", "cost" → `decimal`
- "enabled", "active", "published", "featured", "is_*" → `boolean`
- "tags", "categories", "type", "status" (taxonomy ref) → `entity_reference`
- "components", "paragraphs", "sections", "items" → `entity_reference_revisions`
- "media", "video", "audio" → `entity_reference` (with media target)
- "options", "choices", "select" → `list_string`

If truly ambiguous, default to `string`.

### Widget (if empty)
Use the default widget for the field type:
- `string` → Textfield (string_textfield)
- `string_long` → Text area (multiple rows) (string_textarea)
- `text_long` → Text area (multiple rows) (text_textarea)
- `boolean` → Single on/off checkbox (boolean_checkbox)
- `integer` → Number field (number)
- `decimal` → Number field (number)
- `list_string` → Select list (options_select)
- `list_integer` → Select list (options_select)
- `datetime` → Date and time (datetime_default)
- `daterange` → Date and time range (daterange_default)
- `link` → Link (link_default)
- `image` → Image (image_image)
- `file` → File (file_generic)
- `email` → Email (email_default)
- `entity_reference` → Autocomplete (entity_reference_autocomplete)
  - If target is media → Media library (media_library_widget)
  - If target is taxonomy with few terms → Select list (options_select)
- `entity_reference_revisions` → Paragraphs (stable) (paragraphs)
- `webform` → Webform (webform_entity_reference_select)

Format as: `Label (machine_name)`

### Cardinality (if empty)
- Entity reference fields with "components", "items", "paragraphs", "tags" in name → `Unlimited`
- Image/file fields with plural name → `Unlimited`
- Everything else → `Single`

### Required (if empty)
- Default to `No`
- Title/name fields → `Yes`

### Description (if empty)
- Leave as `-`

### Other (if empty)
- For entity_reference: add `References: <target_bundles>` if known
- For datetime: add `Date only` or `Date and time` as appropriate
- Otherwise: `-`

## Step 5: Complete the title and links

If the title still has `<Label>` or `<number>` placeholders:
- Replace `<Label>` with the bundle label (inferred from the field context or ask the user)
- Replace `<number>` with the ticket number (ask user or use `001`)

If admin URLs have `<admin_url>` or `<machine_name>` placeholders, fill them in based on the entity type and machine name.

## Step 6: Complete permissions (if empty)

If the permissions table has empty rows:
- Add a reasonable default: an "Administrator" row with all Yes, and a "Content Author" or "Content Editor" row with Create/Edit own = Yes, others = No.
- If the user has specified roles, use those instead.

## Step 7: Present changes

Before writing, show the user what you've filled in. List each field and what was auto-completed. Ask for confirmation before saving.

## Important Rules

- **Never overwrite values the user has already filled in.** Only fill empty cells.
- **When uncertain about a field type, ask the user** rather than guessing wrong.
- **Preserve the exact markdown structure** — only modify cell values, don't rearrange the table.
- **Remove the instruction HTML comment** (`<!-- INSTRUCTIONS: ... -->`) after completing the ticket.
- **Keep the entity_type HTML comment** for future reference.
